# diagnose-stocks-first-principles

一个面向股票研究的 Codex skill，用第一性原理把股票分析拉回到几个基本问题：

- 买股票是在买一家公司的部分所有权；
- 企业价值来自未来可分配现金流；
- 好公司不等于好股票，价格和安全边际必须单独判断；
- 投资结论必须区分事实、推断、假设和未知；
- 任何买卖判断都应该是条件化的，而不是一句简单的“买”或“卖”。

## 适合什么场景

适合用来分析或调查公开上市股票，包括：

- A 股、港股、美股、ADR；
- 单只股票的投资备忘录；
- 多只股票的横向比较；
- “这家公司值不值得研究”；
- “现在价格隐含了什么预期”；
- “我已经持有，要不要继续持有/加仓/减仓”的条件化复盘。

它不是用来做短线喊单或技术指标预测的。

## 核心框架

这个 skill 会强制分析者回答：

> 如果我用今天的市值买下整家公司，我未来可能拿到什么现金流？什么会毁掉这些现金流？我为这种不确定性付了什么价格？

分析流程包括：

1. 证券身份确认：ticker、交易所、币种、股本、企业价值。
2. 来源台账：优先使用公司公告、年报、10-K/10-Q、交易所文件等一手来源。
3. 第一性原理拆解：客户需求、收入引擎、单位经济、现金转化、护城河、资本配置。
4. 财务质量检查：利润、自由现金流、SBC、稀释、负债、分部质量、backlog、客户集中度。
5. 三情景估值：Downside / Base / Upside，必须列收入、毛利率、FCF margin、稀释、终值方法和隐含回报。
6. 反向思考：先写熊市情景、失败模式和能证伪投资论点的证据。
7. 条件化结论：输出“继续研究”“观察名单”“质量好但贵”“风险主导”“太难”等判断，而不是无条件买卖指令。

## 使用方式

在 Codex 中安装或引用该 skill 后，可以这样提问：

```text
用 $diagnose-stocks-first-principles 分析一下 RKT.US
```

或者：

```text
用 $diagnose-stocks-first-principles 给 RKLB.US 写一份第一性原理股票诊断报告
```

如果要求保存 Markdown 报告，应使用 `references/output-template.md` 的结构，并在交付前运行：

```bash
python scripts/audit_stock_memo.py path/to/report.md
```

审计脚本会检查报告是否包含来源台账、证据地图、三情景估值、关键催化剂、失败模式、日期、来源链接和免责声明。

## 重要提醒

本 skill 只用于研究和教育，不构成个性化投资建议。股票价格、财报、利率、监管、公司公告和市场环境都会变化；涉及当前事实时必须重新核验最新来源。
